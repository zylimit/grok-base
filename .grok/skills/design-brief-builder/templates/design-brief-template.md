## 完整示例

以下是一个「AI 聊天助手桌面端」的 Design Brief 示例：

```markdown
## 产品概要
- 产品名称：Forge
- 产品类型：Desktop（Electron）
- 目标用户：开发者和技术用户，需要本地运行的 AI 助手
- 核心功能：聊天对话、Agent 管理、IM 桥接、定时任务

## 设计方向

### 情绪关键词
- **专业** — 面向开发者，不要花哨，信息传达清晰
- **沉稳** — 深色主题为主，长时间使用不疲劳
- **精致** — 细节考究但不过度设计，每个像素都有意义

### 参考产品
| 参考产品 | 喜欢的方面 | 不喜欢的方面 |
|---------|-----------|-------------|
| Linear | 信息密度高但不杂乱，动画流畅 | 无 |
| Arc Browser | 侧边栏设计，色彩运用 | 有些功能隐藏太深 |
| Raycast | 快捷键驱动，极简但功能强 | 有时信息太少 |

### 反面参考
- 企业后台风格：不要 Element UI / Ant Design 那种千篇一律的蓝白管理后台
- 花哨动画：不要 Framer 官网那种满屏动效

## 视觉规范

### 色彩方向
- **主题模式**：深色为主，支持浅色切换
- **色彩温度**：冷色
- **品牌主色方向**：深灰为底，靛蓝色作为品牌识别色
- **强调色方向**：珊瑚色用于警告/重要操作
- **品牌资产**：有自定义 mascot 图标

### 信息密度
- **密度方向**：偏紧凑
- **参考基准**：接近 Linear 的密度
- **理由**：产品有聊天、管理、设置等多个功能区，需要高效利用空间

### 排版方向
- **字体气质**：几何无衬线，干净利落
- **中文字体偏好**：系统默认即可
- **标题风格**：与正文差异适中，不需要太夸张

### 交互风格
- **动画程度**：适中——页面切换和列表操作有过渡，不需要花哨效果
- **过渡效果**：快速渐变为主，不要弹跳效果
- **整体节奏**：沉稳但不迟钝

## 核心页面视觉备注

### 聊天界面
- **核心交互**：用户输入消息，AI 流式回复，支持工具调用展示
- **视觉方向**：偏 Claude/ChatGPT 的对话布局，AI 回复支持 Markdown 渲染
- **特殊要求**：工具调用要有可折叠的详情卡片，不能占满屏幕

### 管理面板（Skills/Agents/MCP）
- **核心交互**：三栏布局——分类导航 + 列表 + 编辑器
- **视觉方向**：偏 VS Code 的面板风格，左侧树形导航
- **特殊要求**：编辑器支持代码高亮和预览切换

### 侧边栏导航
- **核心交互**：图标导航 + Workspace 切换
- **视觉方向**：偏 Slack 的侧边栏，可折叠
- **特殊要求**：折叠后只显示图标

## 状态设计
- **空状态**：简短文案引导 + 操作按钮，不用插画
- **加载状态**：AI 回复用打字机光标，其他用骨架屏
- **错误状态**：内联错误提示，红色珊瑚色边框，不用弹窗
```

---

## 写作要点

1. **情绪关键词**：最多 3 个，每个要解释在本产品中的具体含义
2. **参考产品**：必须标注"具体喜欢哪里"，不能只写产品名
3. **色彩方向**：写方向不写具体值（"蓝紫色调"而非"#6366F1"）
4. **信息密度**：必须附带参考基准（"像 XX 的密度"）
5. **核心页面备注**：只写有设计决策空间的页面，纯后端功能不写
6. **状态设计**：空、加载、错误三种状态都要有方向
7. **整体原则**：这份文档是给设计工具和 dev-builder 看的，要具体到能执行，但不要具体到像素级
