# Skill 骨架模板

创建新 Skill 时，复制以下内容，替换 [占位符]，删除不需要的可选 Section。

---

## 必须有的 Section

```markdown
---
name: [skill-name]
description: [一句话描述：什么时候用、做什么、产出什么]
---

[任务]
    [一句话说清楚这个 Skill 做什么。如有多种模式则分别说明。]

[依赖检测]
    Skill 启动时第一步自动执行。

    必需：
    - [前置文件] → 缺失则提示引导操作
    - [系统工具] → 缺失则由 Agent 自主安装

    可选：
    - [可选依赖] → 缺失则标记降级模式继续工作

    安装策略：
    - 必需依赖缺失或版本不满足时，Agent 自主判断安装方式并直接安装
    - 需要用户权限或用户交互时，提示用户操作
    - 可选依赖缺失时，标记降级模式，不阻塞流程

[第一性原则]
    **[原则名称]**：[一句话说明]
    **[原则名称]**：[一句话说明]
    **[原则名称]**：[一句话说明]
    **可验证优先**：能用脚本、测试、hook 检查的规则，不只写在文档里。
    **联网优先**：涉及外部知识时先 WebSearch 确认再动手。

[文件结构]
    ```
    [skill-name]/
    └── SKILL.md
    ```

[工作流程]
    [第一步：XXX]
        [具体操作]

    [第二步：XXX]
        [具体操作]

    [第N步：XXX]
        [具体操作]

[初始化]
    执行 [第一步：XXX]
```

## 推荐有的 Section

```markdown
[输出风格]
    **语态**：
    - [描述这个 Skill 说话的方式]

    **原则**：
    - x [不做什么]
    - v [做什么]

    **典型表达**：
    - "[示例句子 1]"
    - "[示例句子 2]"

[XXX维度清单]
    [根据领域命名：需求维度清单 / 审查维度清单 / 开发规则清单 / ...]
    [列出这个 Skill 需要关注的所有维度或规则]

[XXX策略]
    [根据领域命名：对话策略 / 审查策略 / 开发策略 / ...]
    [描述执行方法论——怎么做]

[压力场景]
    创建或大改 Skill 时使用。每个场景写清：
    - 触发语句：[用户会怎么说]
    - 易错行为：[Agent 没有规则时会怎么错]
    - 期望行为：[Skill 必须强制什么行为]
```

## 按需有的 Section

```markdown
[信息充足度判断]
    收集/分析型 Skill 使用。判断何时信息足够可以输出。

    必须满足：
    - [条件]
    尽量满足：
    - [条件]

[回退策略]
    发布/部署类 Skill 使用。出问题时怎么回退。

[Phase 完成度判断]
    开发类 Skill 使用。Phase 完成的验证标准。

[多模式工作流程]
    如 Skill 有多种执行模式，分别写：
    [工作流程（模式A）]
    [工作流程（模式B）]
```

---

## 规范速查

| 项 | 规范 |
|----|------|
| Skill 名 | kebab-case（如 skill-builder、dev-planner） |
| 目录位置 | .grok/skills/[skill-name]/ |
| 主文件 | SKILL.md |
| 模板文件 | templates/ 子目录（如有） |
| Section 标题 | [标题] 格式 |
| 内容缩进 | 四空格 |
| frontmatter | 只有 name 和 description |
| 语言 | 中文 |
